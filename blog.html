Why I built this

I study bacteriophages, and a lot of my work comes down to one recurring question: where else does this protein family show up? A plain BLAST search misses the divergent members that matter most — the depolymerases, tail fibers, and receptor-binding proteins that have drifted far from anything in the reference databases. Profile HMMs are the right tool for that job, but running one properly means stitching together half a dozen command-line programs: MAFFT to align, HMMER to build and search, Prodigal to call genes, IQ-TREE for a tree, and a pile of glue scripts to hold it together.

That barrier kept the method out of reach for colleagues and students who are excellent biologists but shouldn't have to become shell experts to answer a straightforward question. So I built HMM Discovery: a single web app that walks through the whole pipeline in the browser, orchestrates those tools behind a friendly UI, and — just as importantly — hands back an output package anyone can reproduce.

Profile-HMM discovery normally means gluing together six command-line tools and hoping you remembered every flag. I wanted my lab to run the whole thing from a browser and get reproducible output at the end.

What it does

🧬

Alignment & trimming

Build a multiple sequence alignment from your seed FASTA with MAFFT and clean it up with trimAl — inspect it before you commit.

📐

Profile HMM building

Turn the alignment into a HMMER3 profile HMM and validate self-recovery so you know the model captures your seeds.

🔍

Database search

Search INPHARED and RefSeq viral proteins and genomes with HMMER, translating nucleotide targets via six-frame ORFs or Prodigal.

🏷️

Hit classification

Sort hits into confidence tiers and annotate them with Pfam domains and VOGDB VFAM viral ortholog families.

🧭

Synteny recovery

Pull the five genes upstream and downstream of each placed hit so you can read genomic context, not just a lone match.

🌳

Phylogeny

Place your seeds and new hits on a tree with IQ-TREE to see how the discovered members relate to the family you started with.

📦

Reproducible export

Export TSVs, GFF3, figures, an HTML report, a Methods paragraph, and a reproducibility manifest in one ZIP.

🖱️

No terminal needed

Create a project, check and install tools, browse local files, and run every stage from the web UI — no Python or shell.

How it works

  1. Load your seeds

    Point the app at a curated protein FASTA (nucleotide FASTA, GenBank, and whole folders work too). It creates a project directory to keep everything organized.

  2. Align and build the model

    Run the MAFFT alignment, trim it, and build a HMMER3 profile HMM. A score-calibration step confirms the model recovers its own seeds before you go looking for more.

  3. Search the databases

    Scan INPHARED and RefSeq viral proteins and genomes with HMMER. Nucleotide targets are translated with exhaustive six-frame ORFs or Prodigal, and large files are chunked with seqkit for speed.

  4. Classify, place, and annotate

    Hits are sorted into confidence tiers, annotated against Pfam and VOGDB VFAM, given synteny neighborhoods, and placed on an IQ-TREE phylogeny.

  5. Export a reproducible package

    One click writes hit tables, a presence/absence matrix, synteny GFF3, figures, an HTML summary, Methods text, and a reproducibility manifest into a timestamped ZIP.

Reproducible by design

A discovery run is only as useful as your ability to defend it later. Every export bundles a reproducibility.json manifest and a METHODS_TEXT.txt paragraph that record the tools, settings, database source URLs, and access dates behind the run — so the same analysis can be re-run, cited in a paper, or handed to a reviewer without archaeology. The ZIP is meant to be the artifact you actually keep, not a pile of loose files you have to reassemble. Under the hood the app runs the exact same pipeline/ code as its command-line scripts, so a point-and-click run and an HPC batch job produce comparable results.
